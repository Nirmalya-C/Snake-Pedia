# Contributing to Snake-Pedia

Thank you for your interest in contributing to Snake-Pedia! We welcome contributions from everyone.

## 🌟 How to Contribute

### Reporting Bugs

If you find a bug, please open an issue with:

1. A clear, descriptive title
2. Steps to reproduce the problem
3. Expected behavior vs actual behavior
4. Screenshots (if applicable)
5. Browser and OS information

### Suggesting Enhancements

We love new ideas! To suggest an enhancement:

1. Check if the feature has already been suggested
2. Open an issue with a clear description of the feature
3. Explain why this feature would be useful
4. Provide examples or mockups if possible

### Pull Requests

We actively welcome your pull requests:

1. **Fork the repository** and create your branch from `main`:
   ```bash
   git checkout -b feature/your-feature-name
   ```

2. **Make your changes**:
   - Follow the existing code style
   - Keep changes focused and atomic
   - Write clear, descriptive commit messages

3. **Test your changes**:
   - Ensure the game still works correctly
   - Test in multiple browsers (Chrome, Firefox, Safari, Edge)
   - Verify responsive design on different screen sizes

4. **Commit your changes**:
   ```bash
   git add .
   git commit -m "Add: brief description of your changes"
   ```

5. **Push to your fork**:
   ```bash
   git push origin feature/your-feature-name
   ```

6. **Open a Pull Request**:
   - Provide a clear description of the changes
   - Reference any related issues
   - Add screenshots or GIFs for UI changes

## 📝 Coding Guidelines

### JavaScript

- Use meaningful variable and function names
- Add comments for complex logic
- Keep functions small and focused
- Use ES6+ features where appropriate
- Avoid global variables

### HTML/CSS

- Use semantic HTML elements
- Follow BEM naming convention for CSS classes
- Ensure accessibility (ARIA labels, keyboard navigation)
- Keep styles organized and reusable

### Commit Messages

Follow the conventional commits specification:

- `feat:` - New feature
- `fix:` - Bug fix
- `docs:` - Documentation changes
- `style:` - Code style changes (formatting, etc.)
- `refactor:` - Code refactoring
- `test:` - Adding or updating tests
- `chore:` - Maintenance tasks

Examples:
```
feat: add pause functionality
fix: resolve collision detection bug
docs: update README with new features
```

## 🎯 Development Setup

1. Clone your fork:
   ```bash
   git clone https://github.com/YOUR_USERNAME/Snake-Pedia.git
   cd Snake-Pedia
   ```

2. Create a feature branch:
   ```bash
   git checkout -b feature/amazing-feature
   ```

3. Make your changes and test locally

4. Push changes and create a pull request

  - Sound effects and music

  - Power-ups
  - High score tracking

- **Improvements**:
  - Better AI algorithm
  - Performance optimizations
  - Mobile touch controls
  - Additional themes

- **Documentation**:
  - Code comments
  - Tutorial videos
  - API documentation
  - Translation to other languages

## 🤔 Questions?

If you have questions about contributing, feel free to:

- Review the README and documentation
